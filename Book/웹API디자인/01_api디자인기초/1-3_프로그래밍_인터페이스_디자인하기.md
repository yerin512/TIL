rest api는 http 프로토콜을 이용해서 목표를 표현한다.

어떻게 리소스와 액션들을 식별할 수 있을까? 어떻게 이것을 경로와 gttp메서드로 표현할 수 있을까


기능적 요구사항을 분석한다.
api목표 캔버스로 리소스와 리소스 사이 관계를 식별한다.
api목표 캔버스를 이용해 액션과 액션의 파라미터 그리고 반환값을 식별한다.
